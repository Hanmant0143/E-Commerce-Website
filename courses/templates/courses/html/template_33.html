Module 34: HTML Entities

Introduction to HTML Entities

In HTML, certain characters have special meanings and are reserved for specific purposes. For example, the less-than symbol (<) and the ampersand (&) are used to define HTML tags and entities, respectively. When you want to display these characters as regular text instead of having their reserved functionality, you need to use HTML entities.

HTML entities are codes that represent special characters and symbols. They start with an ampersand (&) followed by a specific keyword and end with a semicolon (;). Using HTML entities ensures that these characters are correctly rendered and helps avoid character encoding issues.

Commonly Used HTML Entities

Here are some commonly used HTML entities and their corresponding characters:

HTML Entity Character Description
&lt; < Less-than symbol
&gt; > Greater-than symbol
&amp; & Ampersand
&quot; " Double quotation mark
&apos; ' Single quotation mark (apostrophe)
&nbsp;   Non-breaking space

There are many more HTML entities available for various special characters, symbols, and accented letters. You can refer to the official HTML documentation for a complete list.

Avoiding Character Encoding Issues

Character encoding issues can occur when special characters and symbols are not properly handled in web documents. If character encoding is incorrect, it may lead to display errors, broken characters, or unexpected behavior in the web page.

To avoid character encoding issues, use appropriate HTML entities for special characters and symbols. Additionally, ensure that the web page's character encoding is correctly set in the tag within the section of your HTML document.

<meta charset="UTF-8">

The "UTF-8" character encoding is widely used and supports a wide range of characters from various languages.

Practical Exercises

  • Create a web page that demonstrates the use of different HTML entities for displaying special characters and symbols.
  • Test the web page on various browsers to ensure proper rendering of the HTML entities.
  • Experiment with different character encodings to observe the effects on character display.
