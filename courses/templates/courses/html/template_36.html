Module 37: HTML Charset

Understanding Character Encoding

Character encoding is the process of mapping characters to numeric codes that can be stored and interpreted by computers. It is essential for representing text in different languages and character sets. The two most commonly used character encodings on the web are UTF-8 and ISO-8859-1 (also known as Latin-1).

The Charset Attribute in HTML

The charset attribute in the meta tag of an HTML document specifies the character encoding used for that document. It helps web browsers interpret and display the text correctly, especially when dealing with special characters and non-ASCII characters found in different languages.

<meta charset="UTF-8">

The above code sets the character encoding of the HTML document to UTF-8, which is the most widely used and recommended encoding for web content.

Setting the Appropriate Character Set for Different Languages

When creating multilingual websites, it is crucial to use the appropriate character set to support the languages used in the content. UTF-8 is recommended because it can handle a vast range of characters from various languages, making it suitable for global content.

To set the character set for a specific language, you can use the lang attribute on HTML elements that contain text in that language. This helps search engines and screen readers understand the language used in the content, contributing to better accessibility and SEO.

<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Multilingual Website</title>
  </head>
  <body>
    <h1>Hello World!</h1>
    <p lang="en">This is the English version of the content.</p>
    <p lang="fr">Ceci est la version fran√ßaise du contenu.</p>
  </body>
