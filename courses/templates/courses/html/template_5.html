Module 6: Understanding HTML Elements

In HTML, an element is constructed with a start tag, content, and an end tag:

<tagname>Content goes here...</tagname>

Let's explore some examples of HTML elements:

<h1>My First Heading</h1> <p>My first paragraph.</p>

The relationship between the start tag, element content, and end tag is illustrated as follows:

Start tag Element content End tag
<h1> My First Heading </h1>
<p> My first paragraph. </p>
<br> none none

It's important to note that some HTML elements don't have content, such as the <br> element, making them empty elements without an end tag.

HTML elements can be nested, meaning an element can contain other elements. All HTML documents are composed of nested HTML elements.

For instance:

<!DOCTYPE html>
<html>
<body>

<h1>My First Heading</h1>
<p>My first paragraph.</p>

</body>
</html>

In this example, <html> serves as the root element, defining the entire HTML document. Inside the <html> element, the <body> element defines the document's body. Within the <body> element, there are two other elements: <h1> for the heading and <p> for the paragraph.

It is crucial to include the end tag for HTML elements. Although some elements might appear to display correctly without it, omitting the end tag can lead to unexpected results and errors.

HTML also features empty elements, such as <br>, which don't require a closing tag and are used to create line breaks.

Remember that HTML tags are not case-sensitive. While the HTML standard doesn't mandate lowercase tags, it is recommended by W3C for consistency and compliance with stricter document types like XHTML.

For additional information about HTML tags and their attributes, refer to our comprehensive HTML Tag Reference.
